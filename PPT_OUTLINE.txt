SLIDE 2 — 研究动机 (Motivation)
================================================================================

【背景】
- 预训练视觉模型百花齐放：CLIP, DINOv2, MAE, SigLIP, ConvNeXt, Data2Vec...
- 每个模型有不同的训练目标、架构、数据
- 自然的想法：把它们融合起来，"博采众长"

【核心问题】
  融合更多预训练模型，性能一定会更好吗？

【直觉 vs 现实】
  直觉：模型越多 → 信息越丰富 → 性能越高
  现实：性能在 4-5 个模型时到顶，之后反而下降！

SLIDE 5 — 实验二：Full-data 实验 — 区分冗余与维度灾难
================================================================================

【目的】区分两个假说：
  假说 A："先升后降"是表征冗余的固有问题
  假说 B："先升后降"是 few-shot 下的维度灾难

【实验】6 模型固定，3 种方法，10-shot vs Full-data

【数据（做成对比柱状图）】
                  10-shot 最优     Full-data 最优
  Concat 赢      3/7 数据集       6/7 数据集
  MoE 赢         2/7              0/7
  Gated 赢       2/7              1/7

【最有说服力的案例 — SVHN 交叉现象（做成交叉图）】
  10-shot:  Concat 22.47% << Gated 33.01%（压缩帮助巨大）
  Full-data: Concat 83.05% >> Gated 77.97%（全量信息反超）

【结论】性能下降是两层因素的叠加：
  ┌──────────────┬──────────────────────────────┬─────────────┐
  │ 因素          │ 描述                          │ Full-data 解决？│
  ├──────────────┼──────────────────────────────┼─────────────┤
  │ 维度灾难      │ 高维特征 + 少量样本 → 过拟合    │ ✅ 能        │
  │ 表征冗余      │ 相似模型提供重复信息            │ ⚠️ 部分缓解   │
  │ 表征噪声      │ 某些模型对特定任务有害          │ ❌ 不能       │
  └──────────────┴──────────────────────────────┴─────────────┘

SLIDE 6 — 理论框架：三个视角统一解释
================================================================================

【理论 1：Platonic Representation Hypothesis (ICML 2024)】
  "不同架构/训练方法的大模型正在趋同到相似的统计表示"
  → 融合趋同的表征 = 复制信息，不是增加信息

【理论 2：Information Bottleneck (Tishby 2015, Kawaguchi ICML 2023)】
  最优表征：max I(Z;Y) s.t. min I(X;Z)
  每加一个模型 → I(X;Z) 线性增长，但 I(Z;Y) 边际递减
  → 存在最优压缩点（4-5 模型），超过后信噪比下降

【理论 3：Bias-Variance-Diversity Decomposition (JMLR 2023)】
  Error = Bias² + Variance - Diversity
  Var_ensemble = ρσ² + (1-ρ)σ²/n
  当 ρ 高（模型趋同），加模型几乎不降低 variance
  → Diversity（低相关性）比 Quantity（多模型）重要得多

配图建议：三个理论用三个图标/图表并排展示

SLIDE 7 — 实验三：CKA 表征相似度分析
================================================================================

【方法】
  CKA (Centered Kernel Alignment, Kornblith ICML 2019)
  度量两个模型表征空间的相似度 ∈ [0, 1]
  - CKA ≈ 0: 完全不同的表征
  - CKA ≈ 1: 几乎相同的表征

【技术细节】
  特征层级：最后一层 patch tokens（非 pooled output）
  PCA 降维：150K → 256 维（避免 d>>N 退化）
  样本数：每数据集 2000 样本

【全局平均 CKA 矩阵（做成热力图）】
              clip   dino    mae   siglip  convnext  data2vec
  clip        1.00   0.24   0.07   0.18    0.19     0.16
  dino        0.24   1.00   0.11   0.34    0.44     0.29
  mae         0.07   0.11   1.00   0.31    0.10     0.32
  siglip      0.18   0.34   0.31   1.00    0.27     0.36
  convnext    0.19   0.44   0.10   0.27    1.00     0.22
  data2vec    0.16   0.29   0.32   0.36    0.22     1.00

【关键发现】
  最互补对：CLIP-MAE = 0.07（训练目标完全正交）
  最冗余对：DINO-ConvNeXt = 0.44（跨架构趋同！）

  → 训练目标 > 架构 决定表征差异
    dino-mae = 0.11（同 ViT，极不同训练目标）
    dino-convnext = 0.44（不同架构，相近视觉理解目标）

SLIDE 8 — 实验四：CKA 选择失败 — "多样性 ≠ 有效性"
================================================================================

【实验】用 CKA 指导模型选择顺序 vs 原始顺序
  原始：clip → dino → mae → siglip → convnext → data2vec
  CKA：clip → mae → ...（每步选 CKA 最低的模型）

【结果（做成对比图）】
  35 次对比：原始赢 25 次，CKA 赢 7 次，持平 3 次
  CKA 选择大幅失败！

【原因分析（核心洞察，大字强调）】

  ┌─────────────────────────────────────────┐
  │  融合收益 = 多样性(Diversity)              │
  │            × 任务相关性(Relevance)          │
  └─────────────────────────────────────────┘

  CKA 只度量了多样性，完全忽略了任务相关性！

【对比表】
  模型   CKA to CLIP   多样性    任务相关性    实际效果
  MAE    0.07(极低)    极高      低(重建导向)   差(-1.56pp)
  DINO   0.24(中等)    中等      高(语义丰富)   好(+提升)

【类比】
  选团队 ≠ 选"跟所有人都不一样的人"
  而是选"跟现有成员互补且本身能力强的人"

SLIDE 9 — 我们的方法：MUMS 框架（数学）
================================================================================

【全称】Marginal Utility-based Model Selection (MUMS)

【核心公式（大字展示）】

  ┌──────────────────────────────────────────────┐
  │                                                │
  │  U(m | S, T) = R(m, T)^α  ·  N(m, S)^β       │
  │                                                │
  │  R = 任务相关性（归一化单模型准确率）             │
  │  N = 1 - avg CKA（新颖性）                      │
  │  α, β = 权衡系数                                │
  │                                                │
  └──────────────────────────────────────────────┘

【为什么用乘法而非加法？】

  经典 MMR (1998) 用加法：Score = λ·Rel - (1-λ)·Sim
  问题：Rel > 0 时，高 diversity 仍可推高分数
  → MAE（不相关但独特）会被错误选中

  我们用乘法：Score = Rel^α · (1-ρ)^β
  ✅ Relevance = 0 → Score = 0（不管 diversity 多高）
  ✅ Novelty = 0 → Score = 0（完全冗余的不选）

  实验验证：MAE 的 Relevance ≈ 0，乘法形式正确给出 Utility ≈ 0

【信息论推导（选择性展示）】
  边际信息增益：
  ΔI(m, S) = I(f_m; Y | f_S)
           ≈ I(f_m; Y) · [1 - I(f_m; f_S)/H(f_m)]
           = Relevance × Novelty

【贪心算法】
  1. S ← {argmax_m R(m, T)}    // 从最相关模型开始
  2. 循环：m* ← argmax U(m | S, T)
  3. S ← S ∪ {m*}

【特殊情况】
  α=0, β=1 → 纯 CKA 选择（实验四已证明失败）
  α=1, β=0 → 纯 relevance 排序
  α=1, β=1 → 平衡 joint selection（我们的默认配置）

【Stage 2：自动子集大小选择 K*】

  Stage 1 只决定排序，Stage 2 决定"选几个模型停下来"

  方法：按 MUMS 顺序逐步加模型，验证集性能不再提升则停止
    if val_acc(S ∪ {m*}) ≤ val_acc(S) 连续 patience 步 → 停止
    K* = argmax_k val_acc(S[:k])

  附加优势（免费过滤）：
    乘法形式让 Relevance=0 的模型 Utility=0 → 自动排除
    如 data2vec（所有任务垫底）在无需验证集时就被过滤

【与 DPP 的联系（可选展示）】
  DPP 核分解：L = diag(q)·S·diag(q)
  q_i = quality, S_{ij} = similarity
  MUMS ≈ DPP 的贪心近似，quality = R(m), S = CKA matrix

SLIDE 13 — 实验七：自动子集选择 — "用几个模型？"
================================================================================

【问题】MUMS 给出排序，但用户还想知道：用几个模型？

【方法】Stage 2 — 验证集早停 (patience=2)
  按 MUMS 顺序逐步加模型 → 验证集性能不再提升则停止

【结果（做成表格，K* vs 实际最优 K）】
  数据集      推荐 K*   实际峰值 K   损失       推荐子集
  Country211  1         1           0          clip
  GTSRB       4         4           0          siglip+clip+convnext+dino
  Pets        4         4           0          dino+clip+siglip+convnext
  SVHN        6         6           0          全部
  EuroSAT     6         6           0          全部
  DTD         6         6           0          全部
  STL10       6         5           0.03pp     全部

  准确率：6/7 (86%)，唯一错误仅 0.03pp

【关键发现】
  Country211：K*=1（单模型 CLIP 最优，任何融合有害）→ 自动识别！
  GTSRB：K*=4（4模型峰值 91.81% > 6模型 91.24%）→ 正确停止
  SVHN/EuroSAT/DTD：K*=6（融合一直有益）→ 不误停

【附加优势：乘法形式的免费过滤】
  data2vec 在所有任务上 Relevance=0 → Utility=0 → 自动排除
  无需验证集就能过滤掉最弱模型

→ MUMS 现在是一个完整方法：输入模型池 → 输出推荐子集

SLIDE 16 — 核心贡献总结
================================================================================

【贡献 1：实验发现】
  - 多模型融合"先升后降"现象的系统研究
  - 区分了三层因素：表征冗余 / 维度灾难 / 表征噪声
  - 证明了数据量决定最优融合策略（few-shot: 压缩 > full-data: 保留）

【贡献 2：CKA 分析】
  - 6 模型 × 7 数据集的完整 CKA 表征相似度图谱
  - 发现训练目标 > 架构 决定表征差异
  - 证明了 "多样性 ≠ 有效性" 这一关键洞察

【贡献 3：MUMS 框架】
  - 首次将 CKA（多样性）× 单模型准确率（相关性）以乘法形式结合
  - 两阶段方法：Stage 1 排序 + Stage 2 自动选 K*
  - 数学推导：信息论 + DPP 联系
  - 实验验证：排序胜率从 20% → 73%，子集选择准确率 86%

【贡献 4：实用指南】
  - 不要默认从 CLIP 开始——从任务最优模型开始
  - MAE/Data2Vec 对分类任务价值有限（乘法形式自动过滤）
  - 用 MUMS 自动确定最优子集大小（1~6 个，因任务而异）

附录 B — MUMS 数学公式速查
================================================================================

【定义】
  模型池：M = {m_1, ..., m_K}
  已选集合：S ⊆ M
  目标任务：T，数据集 D

【任务相关性】
  R(m, T) = (Acc(m, T) - min_Acc) / (max_Acc - min_Acc) ∈ [0, 1]

【成对冗余】
  ρ(m_i, m_j | D) = CKA(f_{m_i}(D), f_{m_j}(D)) ∈ [0, 1]

【集合冗余】
  ρ(m, S | D) = (1/|S|) Σ_{s∈S} ρ(m, s | D)

【新颖性】
  N(m, S | D) = 1 - ρ(m, S | D)

【边际效用】
  U(m | S, T) = R(m, T)^α · N(m, S | D)^β

【信息论推导】
  ΔI(m, S) = I(f_m; Y | f_S)
           = I(f_m; Y) - I(f_m; f_S) + I(f_m; f_S | Y)
           ≈ I(f_m; Y) · [1 - I(f_m; f_S)/H(f_m)]
           = Relevance(m) × Novelty(m, S)

【CKA 公式】
  CKA(X, Y) = ||Y^T X||²_F / (||X^T X||_F · ||Y^T Y||_F)
  X, Y 先中心化：X ← X - 1·mean(X)

【与 DPP 的联系】
  DPP: L = diag(q) · S · diag(q), P(S) ∝ det(L_S)
  MUMS: q_i = R(m_i, T), S_{ij} = CKA(m_i, m_j)
  贪心 MUMS ≈ 贪心 DPP 近似

【特殊情况】
  α=0, β=1 → 纯 diversity（CKA-only）→ 实验证明效果差
  α=1, β=0 → 纯 relevance           → 忽略冗余
  α=1, β=1 → 平衡 joint（推荐）      → 实验验证最优

【两阶段完整算法】

  Stage 1 — Ordering:
    S ← {argmax_m R(m, T)}
    while M \ S ≠ ∅:
        m* ← argmax_{m ∈ M\S} R(m,T)^α · N(m,S|D)^β
        S ← S ∪ {m*}

  Stage 2 — Selection (K* determination):
    best_k ← 1, best_acc ← val_acc(S[:1])
    for k = 2 to |S|:
        acc ← val_acc(S[:k])
        if acc > best_acc:
            best_acc ← acc, best_k ← k
        elif no improvement for patience steps:
            break
    return S[:best_k]
